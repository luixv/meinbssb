❌ Kritische BITV 2.0 Verstöße

📝 BITV 4.1.2 - Name, Rolle, Wert (Level A)

Problem: Passwort-Felder haben keine semantische Kennzeichnung

TextFormField( controller: controller, obscureText: !isVisible, // Fehlende Semantics für Screen Reader )

Auswirkung: Screen Reader können Passwort-Felder nicht korrekt identifizieren

🔒 BITV 1.3.5 - Eingabezweck identifizieren (Level AA)

Problem: Passwort-Felder haben keine autocomplete-Attribute

// Fehlend: autocomplete für Passwort-Manager TextFormField( // autocomplete: "current-password" // autocomplete: "new-password" )

Auswirkung: Passwort-Manager können nicht zwischen aktuellem und neuem Passwort unterscheiden

👁️ BITV 1.4.1 - Verwendung von Farbe (Level A)

Problem: Passwort-Stärke nur durch Farben kommuniziert

LinearProgressIndicator( value: _strength, valueColor: AlwaysStoppedAnimation(_strengthColor(_strength)), ) // Nur Farbe, keine zusätzlichen visuellen Hinweise

Auswirkung: Farbblinde Nutzer können Passwort-Stärke nicht erkennen

🗣️ BITV 4.1.3 - Statusmeldungen (Level AA)

Problem: Keine Live-Region-Ankündigungen für Validierung

String? _validatePassword(String? value) { // Validierungsergebnisse werden nicht angekündigt if (value.length < 8) return 'Mindestens 8 Zeichen'; }

Auswirkung: Validierungsfehler werden nicht automatisch vorgelesen

📱 BITV 2.5.3 - Beschriftung im Namen (Level A)

Problem: Passwort-Sichtbarkeits-Button hat keine aussagekräftige Beschriftung

IconButton( icon: Icon(isVisible ? Icons.visibility_off : Icons.visibility), onPressed: onToggleVisibility, // Fehlende semanticLabel )

Auswirkung: Screen Reader können Funktion des Buttons nicht beschreiben

🎯 BITV 3.3.2 - Beschriftungen oder Anweisungen (Level A)

Problem: Fehlerbehandlung ohne assistive Ankündigungen

void _showPasswordIncorrectSnackbar() { ScaffoldMessenger.of(context).showSnackBar( const SnackBar(content: Text(Messages.currentPasswordIncorrect)), ); // Keine SemanticsService.announce }

Auswirkung: Fehlermeldungen werden nicht von Screen Readern angekündigt

💡 Empfohlene Verbesserungen

1. Passwort-Felder mit Semantics umhüllen

Semantics( textField: true, label: 'Passwort-Eingabefeld: $label', hint: 'Eingabe des Passworts mit Sicherheitsanforderungen', obscured: !isVisible, child: TextFormField( autocomplete: label.contains('Aktuell') ? 'current-password' : 'new-password', // ... ), )

2. Passwort-Sichtbarkeits-Button zugänglich machen

Semantics( button: true, label: '${isVisible ? "Passwort verbergen" : "Passwort anzeigen"}', hint: 'Schaltet die Sichtbarkeit des Passworts um', child: IconButton( icon: Icon( isVisible ? Icons.visibility_off : Icons.visibility, semanticLabel: isVisible ? "Passwort verbergen" : "Passwort anzeigen", ), onPressed: onToggleVisibility, ), )

3. Passwort-Stärke-Anzeige mit Live Region

Semantics( liveRegion: true, label: 'Passwort-Stärke-Anzeige', value: 'Aktuelle Stärke: ${_strengthLabel(_strength)}', child: Column( children: [ Row( children: [ Expanded(child: LinearProgressIndicator(...)), const SizedBox(width: 8), // Zusätzliche Icons für bessere visuelle Unterscheidung Icon(_strengthIcon(_strength), color: _strengthColor(_strength)), Text(_strengthLabel(_strength)), ], ), ], ), )

4. Validierung mit SemanticsService ankündigen

String? _validatePassword(String? value) { final error = _performValidation(value); if (error != null && mounted) { Future.microtask(() { SemanticsService.announce( 'Passwort-Validierungsfehler: $error', TextDirection.ltr, ); }); } return error; }

5. Fehlerbehandlung mit Ankündigungen

void _showPasswordIncorrectSnackbar() { SemanticsService.announce( 'Fehler: Das aktuelle Passwort ist nicht korrekt', TextDirection.ltr, ); ScaffoldMessenger.of(context).showSnackBar( SnackBar( content: Semantics( liveRegion: true, child: const Text(Messages.currentPasswordIncorrect), ), ), ); }

6. Form-Container strukturell kennzeichnen

Semantics( container: true, label: 'Passwort ändern Formular', hint: 'Eingabebereich für aktuelles und neues Passwort', child: Form(key: _formKey, ...) )

7. Lade-Zustand zugänglich machen

FloatingActionButton( onPressed: _isLoading ? null : _handleSave, child: _isLoading ? Semantics( label: 'Passwort wird gespeichert, bitte warten', child: const CircularProgressIndicator(...), ) : Semantics( button: true, label: 'Passwort speichern', hint: 'Speichert das neue Passwort nach erfolgreicher Validierung', child: const Icon(Icons.save), ), )

8. Passwort-Anforderungen als Liste strukturieren

Semantics( readOnly: true, label: 'Passwort-Anforderungen', child: Column( crossAxisAlignment: CrossAxisAlignment.start, children: [ const Text('Passwort-Anforderungen:'), _buildRequirement('Mindestens 8 Zeichen', _strength >= 0.25), _buildRequirement('1 Großbuchstabe', hasUppercase), _buildRequirement('1 Kleinbuchstabe', hasLowercase), _buildRequirement('1 Zahl', hasNumber), _buildRequirement('1 Sonderzeichen', hasSpecial), ], ), )
